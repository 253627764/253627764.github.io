rake generate 生成静态文件 
rake watch 检测文件变化，实时生成新内容
rake preview 在本机4000端口生成访问内容
rake deploy 发布文件

可以在generate后把网站放到本地的apache服务上先看一下效果，没问题再deploy
脚本是：
rake generate
cp -r public/* /Library/WebServer/Documents/

一些定制信息：
1. 定制头文件：
/source/_includes/custom/head.html

2. 定制About me: （介绍的文档：http://octopress.org/docs/theme/template/）
source/_includes/custom/asides/about.html
然后修改 _config.yml

3. 修改_config.yml
     把twitter完全注释掉，否则会连接twitter取js代码。然后被墙，造成访问一直loading

4. 修改：sass/custom/_layout.scss
    增加：$indented-lists: true; 使列表不再往左缩进


5. 增加copyright：
     修改source/_includes/post/copyright.html 文件。里面写上版权信息。
     修改source/_layouts/post.html 文件，在合适位置加上：
     <p class="meta">
          {% include post/copyright.html %}
     </p>

6. 增加分享到新浪微博功能：
     在_config.yml中增加一项： weibo_share: true
     修改 source/_includes/post/sharing.html ，增加：
```
     ｛% if site.weibo_share %｝
         ｛% include post/weibo.html %｝
     ｛% endif %｝
     增加source/_includes/post/weibo.html

本人定制--安装好了ruby和gem等工具要求版本 ruby2.1.1或以上 gem到最新版本 

首先下载代码
1.git clone git://github.com/imathis/octopress.git octopress

2.《定制1》首先修改Gemfile中的rake版本号，由于本人的MAC10.9.2 电脑，Ruby版本为 2.1.1  rake版本为10.3.1
 最终修改为 gem 'rake', '~> 10.3.0'

3.
cd octopress
gem install bundler
bundle install             根据Gemfile文件进行安装特定版本的文件


4.在octopress目录中配置github账号---设置github的账号和仓库地址
rake      setup_github_pages
提示可以用2种方式进行输入         
输入   git@github.com:253627764/253627764.github.io  这种方式需要SSH key，安全性高---尝试OK
或者输入    https://github.com/253627764/253627764.github.io.git   用github账号和密码方式


5.安装默认的Octopress主题
rake install  
rake generate
rake preview 本地验证


6.
将博客发布到Github上，输入下面命令：
rake deploy	测试用https提交成功



7.别忘了把所有源文件发布到 source 分支下面：

注意没有用git init 说明用的是 octopress下面的git
cd octopress
git add .
git commit -m “your message”
git push origin source               提交到远程的source分支


8.绑定域名
cd octopress
echo 'my.seevstudio.com >> source/CNAME

git add .
git commit -m "add CNAME FILE"
git push origin source      #注意是添加到了source分支下
